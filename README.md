# COMP3011 Search Engine Tool

Python command-line search engine for the COMP3011 Coursework 2 target site:
`https://quotes.toscrape.com/`.

The tool crawls the quote pages, builds an inverted index containing word
frequencies and positions per page, saves the index to disk, reloads it later,
prints postings for individual words, and searches for pages containing all
terms in a query using TF-IDF ranked retrieval.

## Architecture & Design Decisions

### Crawler (`crawler.py`)
`WebsiteCrawler` fetches pages using the `requests` library, parses quote
blocks with `BeautifulSoup`, and follows `<li class="next">` pagination links
until the site has no further pages.  All dependencies (HTTP session, sleep
function) are injected at construction time, making it straightforward to swap
them for fakes in tests.  A 6-second politeness delay is enforced between every
successive request.  If a network or HTTP error occurs the crawl stops early and
returns whatever quotes were collected, rather than crashing.

### Inverted Index (`indexer.py`)
`InvertedIndex` stores the index as a nested dict:

```
{ word → { page_url → { "frequency": int, "positions": [int, …] } } }
```

Choosing a dict-of-dicts gives O(1) average-case lookup by word and by page.
All content on a page (quote text, author name, and tags) is concatenated into
a single token stream, so positions are globally ordered across the full page
content.  This matches the standard model used by real search engines and
enables future proximity-based ranking.  The index is persisted to JSON so it
survives across sessions without re-crawling.

### Search Engine (`search.py`)
`SearchEngine` uses **Boolean AND** semantics: a page must contain every query
term to appear in results.  Pages are then ranked by **TF-IDF** score:

```
TF(term, page)  = freq(term, page) / total_tokens(page)
IDF(term)       = log( (N + 1) / (df + 1) ) + 1     [Laplace-smoothed]
Score(page)     = Σ  TF × IDF  for each query term
```

TF-IDF is preferred over raw frequency because it rewards pages where a term
is proportionally more prominent (high TF) while down-weighting terms that
appear on nearly every page (low IDF).  Pages with identical scores are sorted
lexicographically by URL for determinism.

### CLI Shell (`main.py`)
`SearchShell` provides a thin REPL layer that maps user commands to the three
back-end components.  It handles errors from the crawler and file I/O and
returns descriptive strings so that all branching can be tested without
starting a real interactive session.

## Dependencies

- Python 3.11 or later
- `requests` for HTTP requests
- `beautifulsoup4` for HTML parsing
- `pytest` for the test suite

Install all Python dependencies with:

```bash
python -m pip install -r requirements.txt
```

On Windows, if `python` is not available but the Python launcher is installed,
use:

```powershell
py -m pip install -r requirements.txt
```

Create and activate a virtual environment, then install the dependencies.

```bash
python -m venv .venv
source .venv/bin/activate
```

```powershell
.\.venv\Scripts\Activate.ps1
```

## Commands

### `build`

Crawls `https://quotes.toscrape.com/`, waits at least 6 seconds between
successive page requests, builds the inverted index, saves it to
`data/index.json`, and makes it available for searching in the current session.

```text
> build
Built index from 100 quotes across 10 pages and 849 terms.
```

### `load`

Loads a previously saved index from `data/index.json`.

```text
> load
Loaded index from data\index.json.
```

### `print <word>`

Prints the inverted index postings for one word. Lookup is case-insensitive.

```text
> print nonsense
Postings for 'nonsense':
https://quotes.toscrape.com/page/2/ -> frequency=1, positions=[398]
https://quotes.toscrape.com/page/7/ -> frequency=1, positions=[293]
```

If the word is not in the current index:

```text
> print missingword
No postings found for 'missingword'.
```

### `find <query>`

Returns pages that contain all query terms, ranked by TF-IDF score. Queries
are case-insensitive and can contain multiple words.

```text
> find indifference
Results for 'indifference':
https://quotes.toscrape.com/page/2/
```

```text
> find good friends
Results for 'good friends':
https://quotes.toscrape.com/page/2/
https://quotes.toscrape.com/page/6/
```

If no page contains all query terms:

```text
> find wordthatdoesnotexist
No results found for query 'wordthatdoesnotexist'.
```

### `exit`

Closes the interactive shell.

```text
> exit
Goodbye.
```

## Testing

Run the test suite from the repository root:

```bash
python -m pytest -q
```

Windows PowerShell:

```powershell
py -m pytest -q
```

The tests cover:

- crawling and pagination without calling the live website
- the 6-second politeness delay between successive page requests
- graceful handling of HTTP errors during crawling
- tokenisation, case-insensitive indexing, frequencies, and positions
- saving and loading the index
- single-word and multi-word Boolean AND search
- TF-IDF ranking (proportional term density outranks raw frequency)
- shell command validation and end-to-end integration
- error messages for missing index file and crawl failures

## Notes

- The crawler uses a default politeness delay of 6 seconds, matching the
  coursework requirement.
- The index stores each normalised word as a key, then maps page URLs to a
  frequency count and a list of token positions.
- Build the index before using `print` or `find` in a new session, or run
  `load` if `data/index.json` has already been created.
- TF-IDF ranking ensures that pages where query terms are proportionally more
  prominent rank above pages that merely mention them incidentally.
